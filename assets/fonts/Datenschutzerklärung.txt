Welche Daten werden von uns gespeichert und verarbeitet?  

Dieser Abschnitt erläutert die Arten von Daten, die von der Software „Laxout“ erfasst, gespeichert und verarbeitet werden, sowie die Zwecke und Methoden dieser Verarbeitung.  

Die Einhaltung der geltenden Datenschutzgesetze und -vorschriften ist für uns von höchster Bedeutung, und wir setzen uns dafür ein, die Privatsphäre und die Daten unserer Nutzer zu schützen.  

  

1. User UID – Eine Anonymisierte Benutzerkennung
Mit dem Begriff der “User-UID” ist eine zufällig generierte Zeichenfolge für die Identifizierung durch unseren Server gemeint. Bei der Erstellung eines Benutzeraccounts durch einen Therapeuten/BGM wird diese Zeichenkette erzeugt und dem Benutzernamen des Patienten zugeordnet. Diese User-UID wird lokal auf ihrem Gerät und auf unserem Server gespeichert.  

 

Token
Diese Software nutzt zudem eine Token-Authentifizierung, um Daten vor Dritten zu schützen. Token sind ähnlich wie User-Uids und werden ebenfalls auf ihrem Gerät und unseren Servern gespeichert. Aus sicherheitstechnischen Gründen können wir Ihnen jedoch keine näheren Informationen hierzu geben. 

 

LaxCoins
Durch abgeschlossene “Tests” und “Physio Workouts” haben Benutzer die Möglichkeit, sogenannte “LaxCoins” zu sammeln. Hierbei handelt es sich um eine virtuelle Währung, mit der man sich Gutscheincodes für unseren Shop kaufen kann. Die Anzahl dieser Coins wird im Zusammenhang mit dem Benutzerkonto auf unseren Servern gespeichert. 

 

Gekaufte Coupons und Couponcodes:
 Benutzer haben die Möglichkeit, Coupons innerhalb der Anwendung zu erwerben. Die gekauften Coupons sowie die zugehörigen Couponcodes werden in Verbindung mit einer eindeutigen Coupon-ID und dem Benutzerkonto auf unserem Server gespeichert.  

 

Username:
Bei der Erstellung von Benutzerkontos durch einen Physiotherapeuten/BGM muss ein Benutzername angegeben werden, welcher auf unserem Server gespeichert wird und der Identifizierung von Benutzerkonten dient. Zum Schutz der Privatsphäre weißen wir Benutzer unserer Software an, diesen Benutzernamen auf einer anonymen Basis zu wählen. Das bedeutet, dass nur der Therapeut/BGM in der Lage ist seine Patienten zu identifizieren. Wir empfehlen das Format: Vorname + Anfangsbuchstabe des Nachnamens. In unseren Allgemeinen Geschäftsbedingungen §9a finden Sie nähere Informationen hierzu.   

 

Übersprungene Übungen
Die App zeichnet Benutzerverhalten auf, zu welchem auch übersprungene Übungen zählen. Zudem wird aufgezeichnet, wie lange ein Benutzer eine Übung ausführt. Nur die ID der übersprungenen Übung wird in Zusammenhang mit dem Benutzerkonto auf unserem Server gespeichert. 

 

Schmerzlevel
In der “Test” Funktion unserer App werden Benutzer gefragt, wie stark ihre Schmerzen gerade sind. Die Erfassung dieses Wertes erfolgt anonym und es wird nur das Datum der Angabe auf unserem Server gespeichert. 

 

Leistungsindex
In der “Test” Funktion unserer App können User zudem ihre Leistung messen. Hierbei werden alle Kräftigungsübungen aus dem individuellen Workout extrahiert und es wird aufgezeichnet, wie lange bzw. Wie oft man eine Übung schafft. Diese Werte werden jedoch nach Berechnung des Leistungsindexes wieder gelöscht und nur der Leistungsindex ((Wert Übung1+ Wert Übung 2 …. Wert Übung X): Anzahl Übungen) an unseren Server übermittelt. Der Leistungsindex ist personenbezogen und es wird der zugehörige Benutzer und das Datum gespeichert. 

 

Individuelle Workouts
Der Physiotherapeut/BGM kann ihnen ein individuelles Workout aus verschiedenen Übungen zusammenstellen. Die jeweiligen IDs der Übungen werden in Zusammenhang mit dem Benutzerkonto des Patienten auf unseren Servern und lokal auf dem Gerät des Nutzers gespeichert. 

 

Eigene Workouts
In der App haben Patienten die Möglichkeit, sich ein eigenes Workout aus ihren Lieblingsübungen zusammenzustellen. Diese Workouts werden lokal auf dem Gerät des Nutzers gespeichert und es findet keine Übermittlung an den Server statt. Bitte beachten Sie als Patient, dass die Erstellung von eigenen Workouts, sowie die Benutzung unserer Software auf eigene Gefahr erfolgt (Laxout AGBs §2 2b).  

 

Logins
Die App trackt das Öffnen von Laxout und speichert einen Wert an monatlichen Logins anonym in einen monatlichen Zeitraum auf unseren Servern. Es gibt jedoch auch eine andere Art von Logins, welche personenbezogen sind: 

 

Letzter Login
Zusätzlich zu der Anzahl von gesamten monatlichen Logins (11. Logins) speichern wir das letzte Login Datum von Benutzern der Laxout App auf unseren Servern. Dieses Datum ist personenbezogen und wird bei jedem neuen Login in die App überschrieben.  

 

Datum der Usererstellung
Erstellt der Physiotherapeut einen Neuen Benutzer, zeichnet die Software das Datum von diesem Ereignis in Zusammenhang mit dem Benutzerkonto auf unseren Servern auf. 

 

An welchen Tagen Workouts erledigt wurden
Die App speichert das Datum und die Art des Workouts auf unseren Servern, wenn ein Benutzer ein Workout abgeschlossen hat. Dies geschieht im Zusammenhang mit dem Benutzerkonto des Patienten. 

 

Letzter Behandlungstermin
Durch manuelle Eingabe des Physiotherapeuten/BGM wird ein Datum auf unseren Servern gespeichert, welches bei der Eingabe des Therapeuten als “Letzter Behandlungstermin” in Zusammenhang mit dem Patientenkonto gespeichert wird 

 

Anweisung für den Patienten
Des weiteren wird die Anweisung für den Patienten in Zusammenhang mit dem Patientenkonto auf unseren Servern gespeichert.
